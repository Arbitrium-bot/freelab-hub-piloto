Freela'B Hub - piloto com backend

Como rodar localmente
1. Abra o terminal nesta pasta.
2. Rode:
   python server.py
3. App:
   http://127.0.0.1:8801/
4. Admin:
   http://127.0.0.1:8801/admin.html

O que agora e real no backend
- Servidor Python servindo o app e as APIs.
- Banco PostgreSQL em producao quando DATABASE_URL existe.
- Fallback em JSON na pasta data/ apenas para teste local sem PostgreSQL.
- Login com sessao por cookie HttpOnly.
- Senha guardada com hash PBKDF2.
- Cadastro via API.
- Confirmacao de e-mail real por HTTPS/Resend quando RESEND_API_KEY esta configurada, com SMTP como alternativa fora do plano free do Render.
- Recuperacao de senha real por e-mail com token de 1 hora.
- Estado do marketplace salvo no backend.
- Painel admin separado para ver usuarios, verificar e bloquear.
- Pagamentos registrados no backend.
- Checkout preparado para receber links reais do Stripe ou gateway equivalente.
- Geocodificacao preparada por OpenStreetMap/Nominatim para salvar latitude e longitude quando o backend estiver online.

Variaveis criadas pelo render.yaml
- HOST=0.0.0.0
- PYTHON_VERSION=3.11.9
- DATABASE_URL=e criada automaticamente pelo banco freelab-hub-db no render.yaml
- APP_BASE_URL=https://freelab-hub-app.onrender.com
- ADMIN_EMAIL=admin@freelabhub.com
- ADMIN_PASSWORD=defina_uma_senha_forte

Variaveis para adicionar depois no Environment do Render, quando quiser ativar e-mail real e pagamento:
- RESEND_API_KEY=chave_api_do_resend_para_envio_por_https
- RESEND_FROM=Freela'B Hub <onboarding@resend.dev> ou remetente verificado
- SMTP_HOST=servidor SMTP do e-mail
- SMTP_PORT=587
- SMTP_TLS=true
- SMTP_SSL=false
- SMTP_USER=usuario SMTP
- SMTP_PASSWORD=senha SMTP
- SMTP_FROM=e-mail remetente
- STRIPE_PAYMENT_LINK_LAB=link_real_do_plano_lab
- STRIPE_PAYMENT_LINK_FREELANCER=link_real_do_plano_freelancer

Observacao
Para o e-mail real funcionar no Render Free, prefira RESEND_API_KEY e RESEND_FROM, porque o Render Free pode bloquear saida SMTP para smtp.gmail.com nas portas 465/587 com erro Network is unreachable. Depois clique em Save Changes/Manual Deploy. O painel admin mostra o envio configurado quando a configuracao basica existe.

Para pagamento 100% real e necessario configurar uma conta de gateway, criar os links/checkout dos planos e preencher as variaveis no Render. Sem essas chaves, o app registra a tentativa e avisa que falta configurar o gateway.

PostgreSQL no Render
O arquivo render.yaml define o banco freelab-hub-db. Ao criar/sincronizar o Blueprint no Render, se esse banco nao existir, o Render cria o PostgreSQL e injeta DATABASE_URL no web service Python.

Importante sobre falha no Render
O Blueprint agora pede so a senha do admin durante a criacao. As configuracoes de SMTP e Stripe ficam fora do render.yaml para nao bloquear o primeiro deploy se algum segredo estiver vazio.

O projeto antigo era static_site. O Render pode falhar se tentar transformar esse servico estatico em web service Python com o mesmo nome. Por isso o render.yaml agora cria o servico novo freelab-hub-app. Use:
https://freelab-hub-app.onrender.com

Depois que o novo web service estiver funcionando, o servico estatico antigo pode ser removido ou deixado apenas como historico.
